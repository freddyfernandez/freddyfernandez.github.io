Carrito de compras

Se realizaron multiples sesiones a partir de las funciones Action Result,transacciones con SQlServer, Se crearon vistas parciales para la interaccion con los modulos del Catálogo de producto y carrito de compras

Funcionalidad

La estructura del diseño se realizo mediante HTML, CSS, JavaScript y jquery. La usabilidad es responsiva y adicionalmente tiene un diseño amigable para el reporte de ventas e inicio de sesion.

Panel de control

Para el acceso a los datos se realizo el consumo de servicios Api Rest y WCF al conectar con el controlador, aqui se creo el modulo de administración del sistema, en el cual podemos realiziar el Crud y la validadcion de la base de datos

Estructura MVC Strut2

Se implemento la solucion web con los frameworks Struts 2 y MyBatis en forma combinada.usando de la herramientas; Tomcat 9.0,IReport y se aplico la metodología MVC Struts 2. Gestionaremos la interacción con los datos mediante MYSQL,procedimientos almacenados,triggers. Los módulos creados son, la seccion por categorias de los productos, carrito de compras, administración de clientes,trabajadores.socios y reporte de Pedidos.

El diseño incluyen Bootstrap, la gestion de la base de datos mediante sesiones por privilegios de administrador o socio, para actualizar nuestros registros, ademas el uso de Ajax para realizar busqueda por nombre o codigo y el manejo de archivos XML para interconectar las capas de las clases objetos,interfaces,action support y las vistas jsps.

Layouts y Firebase

Manejo de la interface mediante propiedades Layouts en los archivos XML,se hizo uso de clases Activity para el manejo de vistas y controladores, adicionalmente se añadió el material design entre iconos y colores en nuestra aplicación móvil. Se aplicó la metodología por capas. Realizaremos la ejecución con el JDK de JAVA. La base de datos será gestionada por Firebase. La estructura del diseño será manejada archivos XML mediante componentes Layout.

Navegacion

Los módulos creados son: Inicio de Sesión, Creación de Post, Creación Historia, y Edición de Perfil.manipulacion de imageens con la libreria picaso, y eventos para realizar notificaciones.

Almacenamiento y Servicios

Todos los datos son guardados en la nube en Firebase,el registro de de sesion mediante facebook y una seccion para geolocalizar la ubicacion de dispositivos moviles.

Modelamiento DAO

modelamiento de paquetes en vistas,servicios,clases, herencias, interfaces, factories,reutilizacion de librerias, para validar registros,configurar la fecha, consultar la base de datos.

Generacion de reportes basados transact sql,procedimientos almacenados,consultas anidadas, reporte en jasper report, uso de trigger para actualizar tablas de la base de datos.

Modelo MVC

Sistema de Ventas elaborado con el entorno Visual studio,perzonalizable ya que esta programado bajo un paradigma de capas: datos,negocio y presentacion.
